About

Hermes Browser Bridge bridges the gap between automation scripts and real live browser tabs.

Why It Exists

Browser automation is broken. Puppeteer and Playwright spin up heavy headless Chromium instances that miss the actual page state. WebSocket injection scripts fail on GitHub, LinkedIn, ChatGPT, and any site with strict Content-Security-Policy rules. Selenium feels like 2012.

Hermes Browser Bridge takes a different approach: inject a lightweight userscript through Tampermonkey, poll a local relay server over HTTP, and execute commands inside the real browser — the same tab you're looking at right now. No heavy headless browser. No CSP blocks. No flaky WebSocket reconnects.

How It's Different

  • Real browser, real context. You are logged in. Cookies work. JavaScript frameworks render. No emulation gap.
  • CSP bypass. Uses Tampermonkey's privileged GM_xmlhttpRequest and HTTP short-polling instead of WebSocket injection.
  • Zero browser config. Install Tampermonkey, paste the script, done.
  • Portable relay. Single-file Python aiohttp server. No Docker. No cloud dependency.
  • Multi-tab. Each tab auto-registers as an independent client.
